Tensorflow入门记——在C++中使用SavedModelBundle加载模型

发表于 2018-07-28

上一篇已经走通了Tensorflow C++ API的编译和打包流程，接下来就该做一些真正有用的工作了：在C++的server调用Tensorflow的API来加载在python环境中预训练好的模型和参数，对样本进行实时预测。我们现在就开始在Tensorflow的环境中编写供Server端调用的动态库代码和接口。

稍微构思一下，我们的动态库至少要提供以下四个接口：

  • 模型的加载接口：用于一次性把模型加载到内存中
  • 模型的释放接口：从内存中清除模型
  • 预测接口：接受适当类型的输入，适配后得到的Tensor对象或者是序列化后的Example对象输入模型，得到预测结果并返回

总结

这样，我们就完成了C++环境下的Tensorflow模型加载和预测功能。整个过程还是比较简单和直观的，难点主要在于Tensorflow的接口定义有些抽象，有很多概念一眼看上去比较难理解。比如

  • 一个用SavedModel方法导出的Tensorflow模型里到底包含了什么？
  • MetaGraph是什么？
  • kSavedModelTagServe又是什么鬼？
  • 为什么MetaGraph里还有各种Signature？
  • 为什么我只是要加载模型预测一下还要面对这么多看不懂的概念。。

其实如果把这份官方文档https://tensorflow.google.cn/guide/saved_model 仔细看完就会发现上面的概念都有介绍，也能理解这一堆概念只是为了使得导出的模型能够做到平台和语言无关而已。多了一层抽象，但通用性却增加了很多。
